########## Claude Code statusline
##########

# What this is:
#   A custom statusline for Claude Code (~/.claude/statusline.sh) that
#   renders model name, context usage %, session tokens + cost, and an
#   optional monthly usage segment. Dracula color palette.
#
# Layout:
#   Opus 4.7 (1M ctx) | ctx: 42.1K/1.0M (4%) | cur 380.0K tok ($1.25) mon 12.4M tok ($85.30)
#
# Files in this directory:
#   statusline.sh     the script Claude Code invokes per render
#   monthly-usage.sh  background helper -- scans ~/.claude/projects/*.jsonl
#                     and writes ~/.claude/statusline-cache/monthly.json
#                     for the statusline to consume. Kicked off by
#                     statusline.sh when its cache is >5min stale.
#   install.sh        copies both scripts into ~/.claude/ and wires the
#                     statusline into ~/.claude/settings.json via jq
#
# Install on a new machine:
#   git clone <this-dotfiles-repo> ~/.dotfiles    # if not already
#   ~/.dotfiles/claude/install.sh
#
# Requirements:
#   bash, jq (used by both the statusline itself and install.sh)
#
# Monthly usage segment:
#   monthly-usage.sh scans ~/.claude/projects/**/*.jsonl for the current
#   month (UTC), sums tokens, and computes cost from a per-million-token
#   price table baked into the script (opus/sonnet/haiku, standard tier).
#   Update the prices in monthly-usage.sh when Anthropic changes rates.
#   No API key required.
#
#   If you delete monthly-usage.sh, statusline.sh silently omits the
#   monthly segment -- the rest of the statusline still works.
#
# Customizing:
#   Edit ~/.dotfiles/claude/statusline.sh, then re-run install.sh.
#   Colors are 24-bit ANSI in the _fg helpers near the top of the file.
